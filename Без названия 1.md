## 18. Типичные проблемы обучения GAN

### Основные проблемы:

**1. Mode Collapse (коллапс мод)**
- Генератор производит ограниченное разнообразие образцов
- Игнорирует части распределения данных
- Генерирует одинаковые или очень похожие примеры

**2. Дисбаланс генератора и дискриминатора**
- Если D слишком силен → G не получает полезных градиентов
- Если G слишком силен → D не может различать реальное и фейковое

**3. Неустойчивость обучения**
- Осцилляции функции потерь
- Отсутствие сходимости
- Взрывающиеся или исчезающие градиенты

### Методы решения:

**Feature Matching**
- Минимизация расстояния в пространстве признаков
- G стремится совпадать со статистикой реальных данных

**Minibatch Discrimination**
- Учёт различий между примерами в батче
- Помогает избежать mode collapse

**Wasserstein GAN (WGAN)**
```
L_D = E[D(x_real)] - E[D(G(z))] - λ·GP
L_G = -E[D(G(z))]

где GP — gradient penalty для Липшиц-ограничения
```

**Spectral Normalization**
- Нормализация сингулярных значений весов дискриминатора
- Контроль Липшиц-константы

**Progressive Growing**
- Постепенное увеличение разрешения генерируемых изображений

## 19. Метрики оценки генеративных моделей

### 1. Inception Score (IS)

```
IS = exp(E_x[KL(p(y|x) || p(y))])

где:
- p(y|x) — условная вероятность класса для сгенерированного x
- p(y) — маргинальное распределение классов  
- KL — дивергенция Кульбака-Лейблера
```

**Интерпретация:**
- Высокое значение IS → лучшее качество
- Оценивает качество (чёткость объектов) и разнообразие
- Типичные значения: 2-12 для ImageNet

### 2. Fréchet Inception Distance (FID)

```
FID = ||μ_r - μ_g||² + Tr(Σ_r + Σ_g - 2(Σ_r·Σ_g)^(1/2))

где:
- μ_r, μ_g — векторы средних для реальных и сгенерированных данных
- Σ_r, Σ_g — ковариационные матрицы
- Tr — след матрицы
```

**Интерпретация:**
- Низкое значение FID → лучшее качество
- Измеряет расстояние между распределениями в пространстве признаков Inception-v3
- Более робастная метрика, чем IS

### 3. Precision & Recall

```
Precision = |S_real ∩ S_gen| / |S_gen|
Recall = |S_real ∩ S_gen| / |S_real|

где S — множество в пространстве признаков
```

**Интерпретация:**
- Precision — качество (все ли сгенерированные реалистичны?)
- Recall — разнообразие (покрыто ли всё распределение?)

## 20. Переобучение (Overfitting)

### Определение
Модель слишком хорошо запоминает обучающие данные, включая шум и специфические особенности, но плохо обобщает на новые данные.

### Признаки на графиках обучения:

```
Training loss: ↓↓↓ (продолжает уменьшаться)
Validation loss: ↓→↑ (сначала падает, затем растёт)

Разрыв: train_acc >> val_acc
```

**Визуальные индикаторы:**
- Большой разрыв между train и validation метриками
- Validation loss начинает расти при уменьшении training loss
- Training accuracy → 100%, validation accuracy стагнирует

## 21. Методы борьбы с переобучением

### 1. L1/L2 регуляризация

**L1 (Lasso):**
```
Loss_total = Loss_original + λ·Σ|w_i|

Эффект: разреживание весов (feature selection)
```

**L2 (Ridge):**
```
Loss_total = Loss_original + λ·Σ(w_i)²

Эффект: уменьшение величины весов (weight decay)
```

### 2. Dropout

```
При обучении:
y = Dropout(x, p) — обнуляет каждый нейрон с вероятностью p

При инференсе:
y = x·(1 - p) — масштабирование выходов
```

**Принцип:** случайное отключение нейронов → ансамблирование подсетей

### 3. Early Stopping

- Мониторинг validation loss
- Остановка при отсутствии улучшения N эпох
- Возврат к лучшим весам

```python
patience = 10
best_loss = inf
counter = 0

if val_loss < best_loss:
    best_loss = val_loss
    counter = 0
else:
    counter += 1
    if counter >= patience:
        stop_training()
```

### 4. Data Augmentation

Искусственное увеличение датасета через трансформации:
- Геометрические: поворот, отражение, масштабирование
- Цветовые: яркость, контраст, насыщенность
- Шум, размытие, cutout

### 5. Batch Normalization

```
BN(x) = γ·((x - μ_B)/√(σ²_B + ε)) + β

Эффект: стабилизация распределений, регуляризация
```

## 22. Метрики качества для классификации

### Confusion Matrix

```
                Predicted
              Pos      Neg
Actual  Pos   TP       FN
        Neg   FP       TN
```

### Основные метрики

**Precision (Точность)**
```
Precision = TP / (TP + FP)

Интерпретация: Какая доля предсказанных положительных
               действительно положительна?
```

**Recall (Полнота, Чувствительность)**
```
Recall = TP / (TP + FN) = TP / (TP + FN)

Интерпретация: Какую долю положительных объектов мы нашли?
```

**F1-Score**
```
F1 = 2·(Precision·Recall)/(Precision + Recall) = 2TP/(2TP + FP + FN)

Интерпретация: Гармоническое среднее Precision и Recall
```

**F_β-Score** (обобщение)
```
F_β = (1 + β²)·(Precision·Recall)/(β²·Precision + Recall)

β < 1 → больше вес Precision
β > 1 → больше вес Recall
```

### ROC-AUC

**ROC-кривая:** TPR vs FPR при разных порогах

```
TPR = TP/(TP + FN) = Recall (True Positive Rate)
FPR = FP/(FP + TN) (False Positive Rate)

AUC ∈ [0, 1]
0.5 — случайный классификатор
1.0 — идеальный классификатор
```

**Применение:**
- Несбалансированные классы → F1, Precision, Recall
- Важен порог классификации → ROC-AUC
- Общая оценка → Accuracy (только для сбалансированных данных)

## 23. Метрики качества для регрессии

### Mean Squared Error (MSE)

```
MSE = (1/n)·Σ(y_i - ŷ_i)²

Свойства:
- Чувствителен к выбросам (из-за квадрата)
- Единицы измерения: [y]²
- Всегда неотрицателен
```

### Root Mean Squared Error (RMSE)

```
RMSE = √MSE = √[(1/n)·Σ(y_i - ŷ_i)²]

Преимущество: те же единицы измерения, что у y
```

### Mean Absolute Error (MAE)

```
MAE = (1/n)·Σ|y_i - ŷ_i|

Свойства:
- Робастен к выбросам
- Интуитивная интерпретация: средняя абсолютная ошибка
```

### R² (Коэффициент детерминации)

```
R² = 1 - SS_res/SS_tot = 1 - [Σ(y_i - ŷ_i)²]/[Σ(y_i - ȳ)²]

где:
- SS_res — сумма квадратов остатков (residual sum of squares)
- SS_tot — полная сумма квадратов (total sum of squares)
- ȳ — среднее значение y

R² ∈ (-∞, 1]
1   — идеальная модель
0   — модель как среднее
< 0 — модель хуже среднего
```

### Mean Absolute Percentage Error (MAPE)

```
MAPE = (100%/n)·Σ|y_i - ŷ_i|/|y_i|

Интерпретация: средняя процентная ошибка
Проблема: не определён при y_i = 0
```

```python
from torch.utils.data import Dataset

class CustomDataset(Dataset):
    def __init__(self, data, labels, transform=None):
        self.data = data
        self.labels = labels
        self.transform = transform
    
    def __len__(self):
        """Обязательный метод: возвращает размер датасета"""
        return len(self.data)
    
    def __getitem__(self, idx):
        """Обязательный метод: возвращает один элемент по индексу"""
        sample = self.data[idx]
        label = self.labels[idx]
        
        if self.transform:
            sample = self.transform(sample)
        
        return sample, label
```

## 26. Технология создания модели на PyTorch

### Базовая структура nn.Module

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super(SimpleCNN, self).__init__()
        
        # Сверточные слои
        self.conv1 = nn.Conv2d(in_channels=3, out_channels=32, 
                               kernel_size=3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.conv3 = nn.Conv2d(64, 128, kernel_size=3, padding=1)
        
        # Пулинг
        self.pool = nn.MaxPool2d(kernel_size=2, stride=2)
        
        # Полносвязные слои
        self.fc1 = nn.Linear(128 * 4 * 4, 256)
        self.fc2 = nn.Linear(256, num_classes)
        
        # Дропаут
        self.dropout = nn.Dropout(0.5)
        
        # Batch Normalization
        self.bn1 = nn.BatchNorm2d(32)
        self.bn2 = nn.BatchNorm2d(64)
        self.bn3 = nn.BatchNorm2d(128)
    
    def forward(self, x):
        # x: [batch_size, 3, 32, 32]
        
        # Block 1
        x = self.conv1(x)           # [B, 32, 32, 32]
        x = self.bn1(x)
        x = F.relu(x)
        x = self.pool(x)            # [B, 32, 16, 16]
        
        # Block 2
        x = self.conv2(x)           # [B, 64, 16, 16]
        x = self.bn2(x)
        x = F.relu(x)
        x = self.pool(x)            # [B, 64, 8, 8]
        
        # Block 3
        x = self.conv3(x)           # [B, 128, 8, 8]
        x = self.bn3(x)
        x = F.relu(x)
        x = self.pool(x)            # [B, 128, 4, 4]
        
        # Flatten
        x = x.view(x.size(0), -1)   # [B, 128*4*4]
        
        # Fully connected
        x = self.fc1(x)
        x = F.relu(x)
        x = self.dropout(x)
        x = self.fc2(x)             # [B, num_classes]
        
        return x

# Создание модели
model = SimpleCNN(num_classes=10)
print(model)
```

### ResNet блок (пример)

```python
class ResidualBlock(nn.Module):
    def __init__(self, in_channels, out_channels, stride=1):
        super(ResidualBlock, self).__init__()
        
        self.conv1 = nn.Conv2d(in_channels, out_channels, 
                               kernel_size=3, stride=stride, padding=1)
        self.bn1 = nn.BatchNorm2d(out_channels)
        
        self.conv2 = nn.Conv2d(out_channels, out_channels, 
                               kernel_size=3, stride=1, padding=1)
        self.bn2 = nn.BatchNorm2d(out_channels)
        
        # Skip connection
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels, 
                         kernel_size=1, stride=stride),
                nn.BatchNorm2d(out_channels)
            )
    
    def forward(self, x):
        residual = x
        
        out = F.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        
        out += self.shortcut(residual)  # Skip connection
        out = F.relu(out)
        
        return out
```

## 27. Цикл обучения в PyTorch

### Полный цикл обучения

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader

# 1. Инициализация
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = SimpleCNN(num_classes=10).to(device)

# 2. Функция потерь и оптимизатор
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=0.001, weight_decay=1e-4)

# 3. Learning rate scheduler (опционально)
scheduler = optim.lr_scheduler.ReduceLROnPlateau(
    optimizer, mode='min', factor=0.5, patience=5
)

# 4. Training loop
num_epochs = 50
best_val_loss = float('inf')

for epoch in range(num_epochs):
    # ====== TRAINING ======
    model.train()  # Включить режим обучения (dropout, batch norm)
    train_loss = 0.0
    train_correct = 0
    train_total = 0
    
    for batch_idx, (inputs, labels) in enumerate(train_loader):
        # Перенос на GPU
        inputs, labels = inputs.to(device), labels.to(device)
        
        # 1. Обнуление градиентов
        optimizer.zero_grad()
        
        # 2. Прямой проход (Forward pass)
        outputs = model(inputs)
        
        # 3. Вычисление loss
        loss = criterion(outputs, labels)
        
        # 4. Обратное распространение (Backward pass)
        loss.backward()
        
        # 5. Обновление весов
        optimizer.step()
        
        # Статистика
        train_loss += loss.item()
        _, predicted = outputs.max(1)
        train_total += labels.size(0)
        train_correct += predicted.eq(labels).sum().item()
    
    # Средние метрики за эпоху
    avg_train_loss = train_loss / len(train_loader)
    train_accuracy = 100.0 * train_correct / train_total
    
    # ====== VALIDATION ======
    model.eval()  # Отключить dropout, batch norm в режим eval
    val_loss = 0.0
    val_correct = 0
    val_total = 0
    
    with torch.no_grad():  # Отключить вычисление градиентов
        for inputs, labels in val_loader:
            inputs, labels = inputs.to(device), labels.to(device)
            
            outputs = model(inputs)
            loss = criterion(outputs, labels)
            
            val_loss += loss.item()
            _, predicted = outputs.max(1)
            val_total += labels.size(0)
            val_correct += predicted.eq(labels).sum().item()
    
    avg_val_loss = val_loss / len(val_loader)
    val_accuracy = 100.0 * val_correct / val_total
    
    # Learning rate scheduler step
    scheduler.step(avg_val_loss)
    
    # Логирование
    print(f'Epoch [{epoch+1}/{num_epochs}]')
    print(f'Train Loss: {avg_train_loss:.4f}, Acc: {train_accuracy:.2f}%')
    print(f'Val Loss: {avg_val_loss:.4f}, Acc: {val_accuracy:.2f}%')
    
    # Сохранение лучшей модели
    if avg_val_loss < best_val_loss:
        best_val_loss = avg_val_loss
        torch.save({
            'epoch': epoch,
            'model_state_dict': model.state_dict(),
            'optimizer_state_dict': optimizer.state_dict(),
            'loss': best_val_loss,
        }, 'best_model.pth')
        print('✓ Model saved!')

# ====== TESTING ======
model.load_state_dict(torch.load('best_model.pth')['model_state_dict'])
model.eval()

test_correct = 0
test_total = 0

with torch.no_grad():
    for inputs, labels in test_loader:
        inputs, labels = inputs.to(device), labels.to(device)
        outputs = model(inputs)
        _, predicted = outputs.max(1)
        test_total += labels.size(0)
        test_correct += predicted.eq(labels).sum().item()

test_accuracy = 100.0 * test_correct / test_total
print(f'Test Accuracy: {test_accuracy:.2f}%')
```

## 28. Аугментация данных (Data Augmentation)

### Определение
Искусственное увеличение разнообразия обучающих данных через применение различных трансформаций. Улучшает обобщающую способность модели и борется с переобучением.

### Примеры аугментаций для изображений

**Геометрические трансформации:**
```python

transform = transforms.Compose([
    # Случайное отражение по горизонтали
    transforms.RandomHorizontalFlip(p=0.5),
    
    # Случайный поворот на ±15°
    transforms.RandomRotation(degrees=15),
    
    # Случайное масштабирование и кадрирование
    transforms.RandomResizedCrop(size=224, scale=(0.8, 1.0)),

        translate=(0.1, 0.1),
```

```python
def mixup_data(x, y, alpha=1.0):
    lam = np.random.beta(alpha, alpha)
    batch_size = x.size(0)
    index = torch.randperm(batch_size)
    
    mixed_x = lam * x + (1 - lam) * x[index]
    y_a, y_b = y, y[index]
    return mixed_x, y_a, y_b, lam

# Loss для MixUp
```

```python
def cutmix_data(x, y, alpha=1.0):
    lam = np.random.beta(alpha, alpha)
    batch_size = x.size(0)
    index = torch.randperm(batch_size)
    
    # Вырезаем и вставляем патч
    bbx1, bby1, bbx2, bby2 = rand_bbox(x.size(), lam)
    x[:, :, bbx1:bbx2, bby1:bby2] = x[index, :, bbx1:bbx2, bby1:bby2]
    
    # Корректируем lambda по площади
    lam = 1 - ((bbx2 - bbx1) * (bby2 - bby1) / (x.size()[-1] * x.size()[-2]))
    y_a, y_b = y, y[index]
    return x, y_a, y_b, lam
```

### 3. Bayesian Optimization

**Принцип:** Построение вероятностной модели (Gaussian Process) целевой функции и выбор следующей точки на основе acquisition function

```
Acquisition functions:
- EI (Expected Improvement)
- UCB (Upper Confidence Bound)
- PI (Probability of Improvement)

Процесс:
1. Инициализация: случайные точки
2. Построение суррогатной модели
3. Оптимизация acquisition function → следующая точка
4. Оценка целевой функции в новой точке
5. Обновление модели
6. Повторить 3-5
```

- Наиболее эффективен для дорогих функций

- Хорошо работает для 5-20 гиперпараметров

**Недостатки:**
- Сложнее в реализации
- Требует больше вычислений на итерацию

```python
class CustomCNN(tf.keras.Model):
    def __init__(self, num_classes=10):
        super(CustomCNN, self).__init__()
        
        self.conv1 = layers.Conv2D(32, (3, 3), activation='relu')
        self.pool1 = layers.MaxPooling2D((2, 2))
        
        self.conv2 = layers.Conv2D(64, (3, 3), activation='relu')
        self.pool2 = layers.MaxPooling2D((2, 2))
        
        self.flatten = layers.Flatten()
        self.fc1 = layers.Dense(128, activation='relu')

        self.fc2 = layers.Dense(num_classes, activation='softmax')

        x = self.conv1(inputs)
        x = self.pool1(x)
        
        x = self.conv2(x)
        x = self.pool2(x)
        
        x = self.flatten(x)
        x = self.fc1(x)
        
        if training:
            x = self.dropout(x)
        
        return self.fc2(x)
```

### ResNet блок в TensorFlow

```python
class ResidualBlock(layers.Layer):
    def __init__(self, filters, stride=1):
        super(ResidualBlock, self).__init__()
        
        self.conv1 = layers.Conv2D(filters, (3, 3), strides=stride, padding='same')
        self.bn1 = layers.BatchNormalization()
        
        self.conv2 = layers.Conv2D(filters, (3, 3), padding='same')
        self.bn2 = layers.BatchNormalization()
        
        # Skip connection
        if stride != 1:
            self.shortcut = layers.Conv2D(filters, (1, 1), strides=stride)
        else:
            self.shortcut = lambda x: x
    
    def call(self, inputs):
        residual = inputs
        
        x = self.conv1(inputs)
        x = self.bn1(x)
        x = tf.nn.relu(x)
        
        x = self.conv2(x)
        x = self.bn2(x)
        
        x += self.shortcut(residual)
        x = tf.nn.relu(x)
        
        return x
```

```python
import tensorflow as tf

# 1. Подготовка данных
train_dataset = tf.data.Dataset.from_tensor_slices((x_train, y_train))
train_dataset = train_dataset.shuffle(buffer_size=1024).batch(32)

val_dataset = val_dataset.batch(32)

model = CustomCNN(num_classes=10)
loss_fn = tf.keras.losses.SparseCategoricalCrossentropy()
optimizer = tf.keras.optimizers.Adam(learning_rate=0.001)

# 3. Метрики
train_loss_metric = tf.keras.metrics.Mean()
train_acc_metric = tf.keras.metrics.SparseCategoricalAccuracy()

val_loss_metric = tf.keras.metrics.Mean()
val_acc_metric = tf.keras.metrics.SparseCategoricalAccuracy()

# 4. Training step
@tf.function  # Компиляция в граф для ускорения
def train_step(x, y):
    with tf.GradientTape() as tape:
        # Forward pass
        predictions = model(x, training=True)
        
        # Compute loss
        loss = loss_fn(y, predictions)
    
    # Backward pass
    gradients = tape.gradient(loss, model.trainable_variables)
    
    # Update weights
    optimizer.apply_gradients(zip(gradients, model.trainable_variables))
    
    # Update metrics
    train_loss_metric.update_state(loss)
    train_acc_metric.update_state(y, predictions)
    
    return loss

# 5. Validation step
@tf.function
def val_step(x, y):
    predictions = model(x, training=False)
    loss = loss_fn(y, predictions)
    
    val_loss_metric.update_state(loss)
    val_acc_metric.update_state(y, predictions)
    
    return loss

# 6. Training loop
num_epochs = 50

for epoch in range(num_epochs):
    print(f'\nEpoch {epoch + 1}/{num_epochs}')
    
    # Reset metrics
    train_loss_metric.reset_states()
    train_acc_metric.reset_states()
    val_loss_metric.reset_states()
    val_acc_metric.reset_states()
    
    # Training
    for step, (x_batch, y_batch) in enumerate(train_dataset):
        loss = train_step(x_batch, y_batch)
        
        if step % 100 == 0:
            print(f'Step {step}, Loss: {loss:.4f}')
    
    # Validation
    for x_batch, y_batch in val_dataset:
        val_step(x_batch, y_batch)
    
    # Print metrics
    train_loss = train_loss_metric.result()
    train_acc = train_acc_metric.result()
    val_loss = val_loss_metric.result()
    val_acc = val_acc_metric.result()
    
    print(f'Train Loss: {train_loss:.4f}, Train Acc: {train_acc:.4f}')
    print(f'Val Loss: {val_loss:.4f}, Val Acc: {val_acc:.4f}')
```

## Дополнительные формулы и концепции

### Размер выходного тензора Conv2D

```
Output_size = ⌊(Input_size + 2·Padding - Kernel_size) / Stride⌋ + 1

Пример:
Input: 32×32, Kernel: 3×3, Padding: 1, Stride: 1
Output = ⌊(32 + 2·1 - 3) / 1⌋ + 1 = 32×32

Input: 32×32, Kernel: 3×3, Padding: 0, Stride: 2
Output = ⌊(32 + 0 - 3) / 2⌋ + 1 = 15×15
```

### Количество параметров CNN

```
Conv2D:
params = (kernel_h · kernel_w · in_channels + 1) · out_channels
       = (k² · C_in + 1) · C_out

где +1 — bias

Dense:
params = (input_features + 1) · output_features

Пример:
Conv2D(in=3, out=32, kernel=3×3): (3² · 3 + 1) · 32 = 896
Dense(in=128, out=10): (128 + 1) · 10 = 1290
```

### Batch Normalization

```
BN(x) = γ · (x - μ_B) / √(σ²_B + ε) + β

где:
- μ_B = (1/m) Σx_i — среднее по батчу
- σ²_B = (1/m) Σ(x_i - μ_B)² — дисперсия по батчу
- γ, β — обучаемые параметры (scale, shift)
- ε — малая константа для численной стабильности (обычно 1e-5)
```

### Функции активации

```
ReLU: f(x) = max(0, x)
Leaky ReLU: f(x) = max(αx, x), где α = 0.01
ELU: f(x) = x если x > 0, α(e^x - 1) иначе
GELU: f(x) = x · Φ(x), где Φ — CDF нормального распределения
Sigmoid: f(x) = 1 / (1 + e^(-x))
Tanh: f(x) = (e^x - e^(-x)) / (e^x + e^(-x))
Softmax: f(x_i) = e^(x_i) / Σe^(x_j)
```
